# 一、命令行工具（~/bin/）

## 1. capture — 快速捕获

**文件：** `~/bin/capture`  
**语言：** Bash  
**调用：** `cap "内容"` / `cap --link "URL" "标题"` / `cap --file 路径`

### 代码逻辑

```bash
INBOX_DIR="$HOME/Obsidian/MyBrain/Inbox"   # 目标目录
DATESTAMP=$(date +%Y-%m-%d)                 # 当前日期
TIMESTAMP=$(date +%H:%M)                    # 当前时间
```

三种模式：

| 模式 | 触发 | 行为 |
|------|------|------|
| 文本捕获 | `cap "内容"` | 创建 `quick-日期-时间.md`，写入frontmatter + 内容 |
| 链接捕获 | `cap --link "URL" "标题"` | 创建 `标题.md`，记录来源URL |
| 文件导入 | `cap --file 路径` | 读取文件内容，追加到新笔记 |

**关键细节：**
- 文件名自动去除非法字符（`sed 's/[\/:*?"<>|]/-/g'`）
- 每条笔记自动带 YAML frontmatter（created、status、tags）
- 最后异步触发 `gbrain sync`（后台静默，不阻塞终端）

**别名：** `.zshrc` 中 `alias cap='~/bin/capture'`

## 2. checkin — 学习打卡

**文件：** `~/bin/checkin`  
**语言：** Bash  
**调用：** `ci "Day 5" 3 "备注"`

### 代码逻辑

```bash
source /opt/anaconda3/etc/profile.d/conda.sh  # 加载conda
conda activate cv                              # 激活cv环境
python3 ~/cv_learning/tracker.py checkin "$@"  # 调用Python追踪器
```

**为什么需要包装脚本：**
1. tracker.py 需要 cv conda 环境（含 Python 3.10 + OpenCV）
2. 简化命令行：`ci "D-14" 3` vs `conda activate cv && python3 ~/cv_learning/tracker.py checkin "D-14" 3`

**别名：** `.zshrc` 中 `alias ci='~/bin/checkin'`

## 3. git-fix — Git自动修复

**文件：** `~/bin/git-fix`  
**语言：** Bash  
**调用：** 自动（launchd每小时） / 手动 `~/bin/git-fix`

### 代码逻辑

```
1. 检查 .git/HEAD 和 .git/config 是否存在
2. 执行 git status 验证仓库可用
3. 如果健康 → 静默退出（exit 0）
4. 如果损坏 →
   a. rsync 备份所有工作文件（排除 .git）
   b. 删除损坏的 .git
   c. git init + git remote add + git fetch --depth 1
   d. rsync 恢复工作文件
   e. 输出修复日志
```

**为什么需要：** iCloud 同步偶尔会损坏 `.git` 目录中的小文件（HEAD、config），导致 `fatal: not a git repository`。

**集成点：**
- `brain-sync` 别名：先调用 git-fix，再提交同步
- `gbrain-sync` launchd：同步前先调用 git-fix
- `git-health` launchd：每小时自动检查

# 二、学习追踪器（~/cv_learning/）

## 4. tracker.py — 核心打卡系统

**文件：** `~/cv_learning/tracker.py`（382行）  
**语言：** Python 3  
**调用：** `python3 tracker.py <命令> [参数]`

### 架构

```
tracker.py
├── load()              # 读取 ~/.tracker.json
├── save(data)          # 写入 ~/.tracker.json
├── calc_streak(data)   # 计算连续打卡天数
├── checkin(day, ⭐, 备注)  # 打卡命令
├── show_today_tasks()  # 显示今日路线任务
├── find_today_tasks()  # 解析学习路线MD文件
├── status()            # 简要状态
├── dashboard()         # 完整仪表盘
└── week()              # 本周小结
```

### 数据存储

**文件：** `~/.tracker.json`

```json
{
  "days": {
    "2026-06-23": {"day": "D-14", "stars": 3, "notes": "...", "time": "21:45"}
  },
  "streak": 1,
  "total_stars": 3,
  "start_date": "2026-06-23"
}
```

### 核心函数详解

#### `checkin(day_label, stars_done, notes="")`

```
1. 加载现有数据
2. 计算增量（今天已打过卡则只加差值，防止重复打卡刷星）
3. 更新当天记录 + 累计星数
4. 重新计算连续天数
5. 保存
6. 打印结果（连续3/7/30天有特殊鼓励信息）
7. 调用 show_today_tasks() 显示待勾选任务
```

**防刷星逻辑（第34-35行）：**
```python
prev_stars = data["days"].get(today, {}).get("stars", 0)
delta = stars_done - prev_stars  # 只加差值
```

#### `find_today_tasks()`

```
1. 读取 Projects/机器视觉学习路线.md
2. 正则匹配日期范围（如 **6/23 周二**）
3. 找到今天的 D-14 标签
4. 在对应行范围内搜索 - [ ] 任务
5. 返回任务列表 [{content, done, star}]
```

#### `dashboard()`

绘制终端仪表盘：
```
╔══════════════════════════════╗
║  ⚡ 今日行动面板    🔥 3天  ║
╠══════════════════════════════╣
║  ✅ 今日已打卡  ⭐3         ║
║  📋 今日任务:              ║
║     ☐ 任务1               ║
║     ✅ 任务2               ║
╚══════════════════════════════╝
```

### 命令速查

| 命令 | 示例 | 输出 |
|------|------|------|
| `checkin "D-14" 3 "备注"` | `ci "D-14" 3 "学完OpenCV"` | 打卡成功+连续天数+累计⭐+今日任务 |
| `dashboard` | `cv` | 完整仪表盘 |
| `status` | `python3 tracker.py status` | 简要状态 |
| `week` | `python3 tracker.py week` | 本周出勤率 |
