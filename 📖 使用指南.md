# 📖 MyBrain 知识库使用指南

## 系统架构

```
┌─────────────────────────────────────────────────┐
│                 MyBrain 知识库                    │
│                                                 │
│   📝 Obsidian  ←→  🧠 gbrain  ←→  🤖 Claude     │
│   (笔记编辑)       (语义搜索)      (AI 问答)      │
│                                                 │
│   📥 cap 命令    ← 快速捕获想法                   │
│   🔄 brain-sync  ← Git 备份 + 索引同步            │
│   📊 定时任务    ← 每日简报 / 周报 / 论文抓取       │
└─────────────────────────────────────────────────┘
```

---

## 核心命令速查

| 命令 | 作用 | 示例 |
|------|------|------|
| `cap "内容"` | 快速捕捉想法到 Inbox | `cap "VLA模型中action head是关键"` |
| `cap --link "URL" "标题"` | 捕捉链接 | `cap --link "https://arxiv.org/..." "新论文"` |
| `cv` | 学习打卡仪表盘 | `cv` |
| `brain-sync` | Git提交 + 同步索引 | `brain-sync` |
| `brain-health` | 系统健康检查 | `brain-health` |
| `brain-stats` | 知识库统计 | `brain-stats` |
| `gbrain query "问题"` | 语义搜索笔记 | `gbrain query "VLA模型架构"` |
| `gbrain search "关键词"` | 关键词全文搜索 | `gbrain search "视觉感知"` |

## 一天的使用流程

### 🌅 早晨 9:00 — 启动与回顾（3分钟）

```bash
# 1. 查看学习打卡仪表盘
cv

# 输出示例：
# ═══════════════════════════════
# 📊 学习仪表盘
# 🔥 连续打卡: 3 天
# ⭐ 累计星星: 12
# 📅 今日: Day 4 / 84
# ═══════════════════════════════
```

打开 Obsidian → `Cmd+N` 创建今日日记 → 写下今天要学的重点。

> 💡 在 Claude Code 中也可以问：**"根据我最近的笔记，今天应该优先学什么？"**

---

### 🕘 上午 — 深度学习与研究（90分钟）

在 Obsidian 中打开 `Projects/机器视觉学习路线.md`，按计划学习。

遇到新概念时：

1. **立即在 Obsidian 中创建新笔记**（`Cmd+N`）→ 自动存入 `Inbox/`
2. **记下核心要点**
3. **用 `[[双向链接]]`** 连接已有笔记

```markdown
---
tags: [机器视觉, VLA, 概念]
created: 2026-06-23
source: "[[2026年6月23日]] 论文精读"
---

# Dynamic Routing

动态路由是 VLA 模型中替代固定 action head 的新范式...

## 与 Action Head 的区别
- [[action head]] 是静态映射
- 动态路由根据输入动态选择执行路径

## 相关
- [[VLA 模型]]
- [[VLM backbone]]
```

---

### ☕ 午间 — 快速捕获（30秒）

```bash
# 午饭时突然想到的灵感，掏出手机/电脑：
cap "动态路由和CKA分析可以结合起来，看看VLA各层冗余度"
```

这条笔记自动出现在 Obsidian `Inbox/` 中，晚上一起整理。

---

### 🕑 下午 — 代码实践

在 `~/cv_learning/week04/` 写代码时，想搜索之前记的概念：

```bash
# 终端直接搜索笔记
gbrain query "图像处理 阈值分割"

# 结果：
# [0.89] 图像处理基础 — 阈值分割、边缘检测...
# [0.76] 基础语法 — OpenCV threshold 用法...
```

也可以直接在 Claude Code 中问：
> "我笔记里记录的 OpenCV threshold 函数怎么用？"

---

### 🌆 傍晚 18:00 — AI 辅助整理（5分钟）

在 Claude Code 中说：

> "帮我分析今天 Inbox 里新增的笔记，给每篇生成3句话摘要和建议的 PARA 分类"

Claude Code 会通过 gbrain MCP 搜索你今天的新笔记，返回类似：

```
📄 Dynamic Routing
   摘要: 动态路由是VLA模型中新兴的动作选择范式...
   建议分类: Resources/文献
   建议标签: [机器视觉, VLA, 概念]

📄 视觉感知
   摘要: 探讨人类视觉感知机制与计算机视觉的交叉...
   建议分类: Areas/心理学
   建议标签: [心理学, 机器视觉, 跨领域]
```

---

### 🌙 晚间 21:00 — 收尾同步（2分钟）

```bash
# 一键完成：Git 备份 + gbrain 索引更新
brain-sync

# 查看知识库状态
brain-stats

# 检查系统健康
brain-health
```

打开 Obsidian `🏠 Home` 页，查看 Dashboard：
- 📥 **Inbox 积压**：今天新捕捉了几条？
- 🏝️ **孤岛笔记**：有没有新增的零链接笔记？
- 🔥 **链接度 Top 10**：哪些笔记成为了知识网络的核心？

---

### 🗓️ 周日 20:00 — 每周回顾

系统会自动生成周报到 `Weekly/` 文件夹。

也可以手动运行：

```bash
bash ~/Obsidian/MyBrain/scripts/weekly-review.sh
```

周报内容：
- 本周新建/修改了哪些笔记
- 知识库统计数据
- 孤立笔记（需要建立链接的）
- Inbox 清空情况

## PARA 工作流

```
📥 Inbox（随时捕获）
    │
    │  每周日整理（20分钟）
    ↓
┌──────────────────────────────────┐
│  🚀 Projects   有截止日期的事      │
│  🌱 Areas      持续关注的领域      │
│  📚 Resources  参考资料和笔记      │
│  📦 Archives   已完成/不再活跃     │
└──────────────────────────────────┘
```

### 整理决策树

```
这条笔记是？
├─ 有截止日期、需要行动？ → 🚀 Projects
├─ 长期关注、没有终点？   → 🌱 Areas
├─ 纯参考资料？           → 📚 Resources
├─ 已经不用了？           → 📦 Archives
└─ 还不确定？             → 📥 留在 Inbox
```

## 定时任务

| 任务 | 时间 | 作用 |
|------|------|------|
| gbrain 索引同步 | 每30分钟 | 自动同步 Obsidian 变更到搜索引擎 |
| 每日简报 | 工作日 9:00 | 学习仪表盘 + 知识库状态 |
| 每周回顾 | 周日 20:00 | 生成周报到 `Weekly/` |
| arXiv 论文抓取 | 每日 7:00 | 最新 CV 论文存入 `Inbox/` |

管理命令：

```bash
# 查看所有定时任务
launchctl list | grep xiaoming

# 加载/卸载
launchctl load ~/Library/LaunchAgents/com.xiaoming.xxx.plist
launchctl unload ~/Library/LaunchAgents/com.xiaoming.xxx.plist
```

## 目录结构

```
~/Obsidian/MyBrain/
  📥 Inbox/          ← 快速捕获入口
  🚀 Projects/       ← 有截止日期的项目
  🌱 Areas/          ← 持续关注的领域
  📚 Resources/      ← 参考资料（论文、代码、笔记）
  📦 Archives/       ← 已归档内容
  📅 Daily/          ← 每日笔记
  📋 Templates/      ← 笔记模板
  📊 Weekly/         ← 自动生成的周报
  🔧 scripts/        ← 自动化脚本
  🏠 Home.md         ← 仪表盘主页
  📖 使用指南.md      ← 你正在看的这个文件
```
