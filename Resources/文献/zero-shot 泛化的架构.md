Zero-shot 泛化并非依赖单一特定架构，而是‌**大规模预训练基础模型（Foundation Models）**‌结合‌**统一任务建模框架**‌的产物。其核心在于通过海量数据学习通用表征，使模型无需针对新任务微调即可推理。‌‌

### 核心架构范式

- ‌**Encoder-Decoder 统一架构**‌：以 T5/mT5 为代表，将各类 NLP 任务转化为“文本到文本”生成问题。编码器深度解析输入语义，解码器基于预训练语言知识重构输出，这种解耦设计使模型能跨领域迁移通用语言模式 。
- ‌**多模态对齐架构**‌：以 CLIP、Youtu-VL 为代表，采用双塔或融合 Transformer 结构，在预训练阶段通过对比学习对齐图像与文本嵌入空间。模型学会将视觉特征映射到语义空间，从而通过文本指令直接识别未见过的类别或执行新任务 。
- ‌**视觉自回归架构**‌：以 VAR、GPT 类视觉模型为代表，将图像离散化为 Token 序列，通过“预测下一个视觉词元”学习数据分布。这种架构赋予模型类似 LLM 的上下文泛化能力，仅需改变推理时的掩码或条件即可实现图像修复、编辑等零样本任务 。
- ‌**世界模型与具身架构**‌：以 DreamZero 为代表，利用视频作为世界演化表征，通过“Slow-Fast”时序建模学习物理规律。此类架构强调隐式规划能力，使智能体能在未见过的环境或任务中实现跨具身迁移 。‌‌

### 实现零样本泛化的关键机制

1. ‌**语义嵌入空间对齐**‌：模型不记忆具体样本，而是学习类别属性、词向量等语义信息，通过对比学习使不同模态数据在共享空间中距离相近，从而识别未接触对象 。
2. ‌**层次化特征表示**‌：底层捕捉通用单元（如词汇、边缘），高层抽象语义与语用规则。这种分层结构确保模型在分布外场景下仍能复用底层通用特征 。
3. ‌**动态提示与上下文推理**‌：通过动态提示合成器或 In-context Learning，将任务意图编码为结构化提示，引导预训练模型激活相关知识路径，无需更新权重 。‌‌

简言之，Zero-shot 泛化的架构本质是‌**“大规模预训练 + 统一接口 + 语义对齐”**‌，其能力边界取决于预训练数据的广度与模型对通用规律的抽象程度。
