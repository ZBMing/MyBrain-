**“中值滤波”** 是我们上一轮讨论的“椒盐噪声”的**标准解法**，也是图像处理中一个非常经典的非线性滤波技术。

结合 GelSight 和 TacForeSight 的上下文，我为你彻底拆解中值滤波是什么、它的工作原理、为什么它对椒盐噪声特别有效，以及在 TacForeSight 系统中的角色。

---

### 🎯 第一层：最直白的定义

**中值滤波（Median Filtering）** 是一种**非线性图像平滑技术**，它将图像中**每个像素的值替换为该像素邻域内所有像素值的中位数**。

**一句话通俗理解**：
中值滤波就是 **“用周围邻居的‘中间值’来替换当前像素”**——就像你问周围 8 个邻居“你们家今天吃了什么”，然后取这些答案的“中间值”作为你自己的答案，避免被某个邻居的“今天吃了火锅”（异常值）带偏。

**一个生活类比：邻居讨论会**

-   假设你住在一个 3×3 的小区里（9 户人家）。
-   其中 8 户人家每天吃米饭，1 户人家今天吃火锅（“异常值”）。
-   **均值滤波**：取 9 户人家食物的“平均值”——这不好定义，而且火锅会被“平均”成半米饭半火锅，很奇怪。
-   **中值滤波**：把这 9 个答案按“奇怪程度”排序（米饭、米饭、米饭、米饭、米饭、米饭、米饭、米饭、火锅），取**中间那个**——米饭。这样就把火锅这个“异常值”完美排除了。

---

### ⚙️ 第二层：中值滤波的核心工作原理

| 步骤 | 操作 | 示例（3×3 邻域） |
| :--- | :--- | :--- |
| **1. 选择邻域** | 对每个像素，取其周围一个**窗口**（如 3×3、5×5）内的所有像素值 | `[10, 12, 11, 9, 255, 10, 13, 12, 11]` |
| **2. 排序** | 将窗口内的像素值按**从小到大**排序 | `[9, 10, 10, 11, 11, 12, 12, 13, 255]` |
| **3. 取中位数** | 取排序后的**中间值**（即第 5 个值） | `11` |
| **4. 替换** | 将当前像素的值替换为中位数 | 当前像素值从 `255`（椒盐噪声）变为 `11` |

**一个关键特性**：中值滤波**不是线性操作**（它不依赖于加权求和），因此它不会像均值滤波那样**模糊边缘**——它能够有效地**去除孤立的异常值**，同时**保持图像的边缘和细节**。

---

### 🔬 第三层：为什么中值滤波对椒盐噪声特别有效？

这是我们上一轮讨论的核心问题——椒盐噪声是**孤立的极端值**（0 或 255），而中值滤波对这类噪声有天然的优势：

| 噪声类型 | 中值滤波的效果 | 原因 |
| :--- | :--- | :--- |
| **椒盐噪声（孤立极端值）** | **完美去除** | 椒盐噪声是**孤立的**——窗口内大多数像素是正常的，异常值（0 或 255）在排序后会被“挤”到序列的两端，中位数不受影响 |
| **高斯噪声（所有像素小幅波动）** | **效果有限** | 高斯噪声影响**所有像素**，中位数虽然有一定抑制作用，但不如均值滤波或高斯滤波有效 |
| **边缘/细节** | **保留良好** | 中值滤波不会像均值滤波那样“模糊”边缘，因为中位数是一个统计量，不会“平均”掉尖锐的变化 |

**一个关键对比**：

| 滤波方法 | 对椒盐噪声的效果 | 对边缘的保持 | 计算复杂度 |
| :--- | :--- | :--- | :--- |
| **均值滤波** | 差（椒盐噪声会被“平均”到邻域中，留下灰色模糊） | **差**（模糊边缘） | 低 |
| **高斯滤波** | 差（椒盐噪声会被“扩散”到邻域） | 中等 | 中等 |
| **中值滤波** | **极好**（孤立的极端值被中位数替换，完美去除） | **好**（保持边缘清晰） | 中等（需要排序） |

### 🔄 第四层：中值滤波在 TacForeSight 中的角色

回到 TacForeSight，中值滤波虽然没有被明确写入论文，但它在系统的数据预处理中可能扮演了重要角色：

| 层面 | 中值滤波的作用 | 具体体现 |
| :--- | :--- | :--- |
| **数据预处理** | 在训练数据送入编码器之前，先进行中值滤波**去除椒盐噪声** | 如果 GelSight 传感器存在坏点或信号干扰，中值滤波可以**在输入端就消除异常像素**，避免噪声污染隐状态 \( z_t \) |
| **数据增强** | 在训练过程中，**随机添加椒盐噪声 + 中值滤波**可以作为数据增强策略 | 让模型学会识别和抵抗椒盐噪声，提高泛化能力 |
| **推理阶段** | 实时推理时，对输入图像进行中值滤波 | 保证 TacForceWM 接收到的是**干净的触觉图像**，提高预测精度和鲁棒性 |

**一个关键问题**：为什么中值滤波没有出现在论文的结构图中？

因为中值滤波属于**数据预处理环节**，而不是**网络结构的一部分**。论文通常只描述网络结构，而预处理（如标准化、去噪、裁剪）属于**实验设置或数据准备**部分，不一定显式出现在网络架构图中。

### 🆚 第五层：中值滤波 vs. 其他滤波方法（在 GelSight 中的选择）

在 GelSight 的图像预处理中，为什么中值滤波是首选？

| 滤波方法 | 在 GelSight 中的效果 | 优缺点 |
| :--- | :--- | :--- |
| **中值滤波** | 去除椒盐噪声，保持形变边缘 | **优点**：保留边缘、有效去除孤立异常值<br>**缺点**：对大窗口计算量稍大 |
| **高斯滤波** | 平滑整体噪声，但会模糊形变细节 | **优点**：有效抑制高斯噪声<br>**缺点**：模糊边缘，可能降低形貌重建的精度 |
| **均值滤波** | 去除随机噪声，但严重模糊边缘 | **优点**：计算简单<br>**缺点**：边缘模糊严重，不适合 GelSight |
| **双边滤波** | 去噪同时保留边缘 | **优点**：结合空间和像素值差异，保留边缘<br>**缺点**：计算量大，参数调优复杂 |

**一个关键结论**：在 GelSight 的预处理中，**中值滤波通常优先于高斯滤波**，因为 GelSight 图像的边缘（形变边界）对后续的光度立体法和三维形貌重建至关重要——模糊边缘会导致形貌重建精度下降。中值滤波在去除椒盐噪声的同时，能够较好地保留这些边缘。

---

### 💡 第六层：中值滤波在 TacForeSight 中的“隐式角色”

虽然论文中没有明确提到中值滤波，但它是 TacForeSight 系统能够**在真实传感器噪声环境下保持鲁棒性**的一个隐含因素：

| 隐含作用 | 具体表现 |
| :--- | :--- |
| **保护三维形貌重建** | 中值滤波避免了椒盐噪声被错误地重建为“形貌尖刺”，保证了光度立体法的输入图像质量 |
| **稳定隐状态编码** | 编码器输入的是干净图像，隐状态 \( z_t \) 不会因椒盐噪声产生不合理的“尖峰” |
| **提高预测精度** | 干净的多帧图像序列有助于 GRU 学习**真实的形变演化规律**，而不是被噪声扰动的序列 |
| **增强实际部署的鲁棒性** | 在实际部署中，传感器可能会遇到不同程度的像素异常或电磁干扰，中值滤波提供了**第一道防线** |

---

### 💎 一句话总结

**中值滤波**是一种非线性图像平滑技术，通过将每个像素替换为其邻域内像素值的中位数，能够**有效去除孤立的极端值（如椒盐噪声）**，同时**保持图像的边缘和细节不被模糊**。在 GelSight 触觉感知中，中值滤波是预处理阶段的重要环节——它能够消除传感器坏点或传输干扰带来的椒盐噪声，保护光度立体法对三维形貌的精确重建。在 TacForeSight 系统中，中值滤波**不属于网络结构本身**，而是属于数据预处理层面，它在系统输入端提供了**第一道噪声防御**，确保编码器接收的是干净的触觉图像，从而保障隐状态 \( z_t \) 的质量和 TacForceWM 的预测精度。
