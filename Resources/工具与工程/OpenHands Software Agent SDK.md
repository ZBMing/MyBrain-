在 AI 智能体开发领域，**OpenHands Software Agent SDK** 指的是 OpenHands V1 版本中，一个**用于构建、运行和部署软件开发 AI 智能体的标准化、可组合的 Python 开发套件**。

可以把它理解为一个“AI 程序员的开发底座”，它统一了智能体的核心逻辑、工具调用、执行环境和交互界面，让开发者能更轻松地创建和扩展自己的 AI 编程助手。

下面我从它的定位、架构设计和它能做什么来帮你梳理一下。

### 🎯 SDK 的核心定位：不只是“库”，而是一个“引擎”

它和我们常说的传统“库”有些不同。你可以这样理解：
*   **传统库（如 OpenAI SDK）**：更多是提供**连接大模型 API 的客户端**，是工具包。
*   **OpenHands Agent SDK**：它是一个**完整的软件智能体引擎**。它不仅封装了模型调用，还集成了代码执行沙盒、文件编辑、状态管理、安全控制等一系列构建智能体所需的**完整底层设施**。

它是 OpenHands 从 V0 版单体架构向 V1 版模块化架构演进的核心成果。

### ⚙️ 它长什么样？—— “四包”模块化架构

为了兼顾灵活性和生产环境的可靠性，这个 SDK 采用了一种“四包”模块化架构，你可以按需组合使用：

| 包名 (Package) | 功能定位 | 核心价值 |
| :--- | :--- | :--- |
| **`openhands.sdk`** | 核心框架 (必选) | 包含 Agent 推理循环、会话状态管理、LLM 接口、工具系统抽象、事件模型等基础组件。是必须的基础。 |
| **`openhands.tools`** | 预置工具集 (可选) | 提供开箱即用的智能体能力，比如执行 Bash 命令、读写编辑文件等。 |
| **`openhands.workspace`** | 执行环境扩展 (可选) | 提供**沙盒环境**的实现，例如 Docker 容器或远程 API 工作区，用于安全地执行智能体代码。 |
| **`openhands.agent_server`** | 远程服务部署 (可选) | 一个基于 FastAPI 的 HTTP/WebSocket 服务器，支持将智能体作为后端服务部署，实现多用户、分布式的远程执行。 |

### 🚀 两大运行模式：从本地原型到生产部署

得益于这种模块化设计，一套 Agent 代码可以无缝地在两种模式下运行，只需**替换工作区（Workspace）类型**即可：
1.  **本地开发模式**：安装 `openhands-sdk` 和 `openhands-tools` 即可。所有组件在**单个进程**内运行，适合快速原型开发和本地实验。
2.  **生产/沙盒模式**：安装全部四个包。智能体运行在 `RemoteWorkspace` 中，可以自动在 Docker 容器内启动 `agent_server`，实现**安全的沙盒隔离**和**多用户并发支持**，是生产级部署的理想选择。

### 💎 它对我们讨论的价值

在《GeneralVLA-2》这类论文的背景下，**OpenHands Software Agent SDK** 代表了当前 AI 编程智能体领域最前沿的工程化方向。它通过清晰的分层设计解决了智能体从研究原型到生产落地的诸多难题（如状态管理、沙盒隔离、多模型路由、安全审计等）。论文中提到的在 **SWE-bench Verified** 上高达 **72%** 的任务解决率，正是基于这套 SDK 构建的智能体所取得的成果。这也解释了为何它会成为评估受控长期记忆系统在复杂软件工程任务中效果时，被提及的重要工具或框架。
